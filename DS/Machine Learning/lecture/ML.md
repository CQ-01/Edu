# 인공지능
> 인공장치들의 지능을 설계하는 것
> <br/>
> \* 지능 : 인간이 행하는 지적 작업의 주체
## 역사
- 1943(여명기) : 두뇌 논리회로 모델링
- 1956(태동기) : 다트머스 회의에서 AI용어 탄생
- 1956 ~ 70(1차 인공지능 붐) : 수동적 대화시스템(지능은 기호처리)
- 1971 ~ 79(1차 빙하기) : 프레임(고려해야할 범위)의 문제
- 1980 ~ 95(2차 인공지능 붐) : 전문가 시스템 활용(지능은 지식)
- 1996 ~ 2009(2차 빙하기) : 지식 획득 병목의 문제
- 2010 ~ 현재(3차 인공지능 붐) : 빅데이터와 딥러닝(지능은 학습)

## 종류
- 약한 인공지능(ANI) : 많은 양의 데이터를 처리하여 특정기능만 수행
- 강한 인공지능(AGI) : 사람처럼 생각하고 판단하는 범용 인공지능
- 특이점(singularity) : 인간과 인공지능 사이의 임계점
- 슈퍼 인공지능(ASI) : 특이점을 지나 인간의 지능을 넘어선 인공지능

## 튜링 테스트
- 컴퓨터가 지능을 가지고 있는지 여부를 조사
- 인간과 컴퓨터에게 질문을 하여 인간의 답과 컴퓨터의 답을 구분할 수 없으면 지능을 갖고 있는 것

# 모델 검증
> 과적합이나 과소적합을 방지 및 모델 학습의 적합도를 알아보기 위해 훈련 데이터 일부를 떼어 검증을 위한 데이터로 사용하는 것
## 오차행렬(confusion matrix)
- 분류에서 가장 많이 사용됨
- 예측한 P/N대비 실제 P/N 현황
  
|TP|FN|
|--|--|
|FP|TN|
- 정확도(accuracy) : 전체 대비 P/N을 정확히 예측한 비율
- 정밀도(precision) : Positive로 예측한 인스턴스 중 정확히 판별한 비율
- 재현율(recall) : 실제 클래스가 Positive인 인스턴스 중 정확히 판별한 비율
## 정밀도와 재현율의 trade-off
- positive 판별기준을 빡빡하게 잡으면(임계값을 크게 잡으면) 정밀도가 올라가고
- positive 판별기준을 널널하게 잡으면(임계값을 작게 잡으면) 재현율이 올라간다
- 상황에 따라 정밀도나 재현율의 중요성이 달라진다
- 재현율 정밀도 그래프
- ![재현율 정밀도 그래프](https://img1.daumcdn.net/thumb/R1280x0/?fname=http://t1.daumcdn.net/brunch/service/user/34ES/image/tFzrPjRTTQ6AzJG4o6HcKu0enHM.png)
## F-score(F-measure)
> 정밀도와 재현율 수치를 한번에 보기 위해 만들어짐
- F1-Score : 일반적으로 사용됨, 정밀도와 재현율의 조화평균
- Fb-Score : 특정수치가 부각되어야 할때 사용됨, 한쪽 수치에 가중치 b처리

## AUC(Area Under Curve)
> 분류 적합도 판단 기준, ROC 곡선 아래의 넓이
- ROC곡선
  - FPR(false positive rate)을 X축으로, TPR(true positive rate)의 그래프
  - FPR : 실제 N클래스인 인스턴스를 P클래스로 잘못 분류한 비율
  - TPR : 실제 P클래스인 인스턴스를 정확히 분류한 비율 == 재현율
-![ROC curve](https://velog.velcdn.com/images/zxxzx1515/post/a1114a22-b77a-4325-a02d-cb41ab079312/image.png)

## 좋은 평가방법
- 모든 상황에 최적화된 모델은 없다
- AUC나 F-Score처럼 범용적으로 사용되는 수치만 있을 뿐
- 많은 수치를 측정해보고, 가장 적정하거나 평균적으로 높게 평가되는 모델을 찾아야 함

# 회귀 분석
> 종속 변수와 독립 변수 간의 관계에 대해 설명하는 통계적 방법. 일반적으로 그래프로
[그래프](https://www.tibco.com/sites/tibco/files/media_entity/2020-09/regression-analysis-diagram.svg)
- 구하고자 하는 종속 변수에 영향을 줄 수 있는 독립변수의 수가 많을 경우 측정 가능한 모든 변수를 정렬하고 어떤 변수가 영향을 미칠지 논리적으로 나타냄
## 회귀선
- 종속 변수와 독립변수 간의 관계를 보여주는 선
- x축 : 독립변수, y축 : 종속변수
- y = wx + b

## 다중회귀
- 회귀당 종속변수는 하나만 존재하나, 독립변수는 여럿 존재 가능 $\rightarrow$ 다중회귀
- 변수간의 복잡한 관계 식별

## 오차항
> 회귀분석 공식을 신뢰할 수 있는 확실성을 보여주는 수치
- 회귀분석은 인과관계가 아닌 변수간의 관계만 예측하므로 독립변수는 종속변수의 완벽한 예측변수가 될 수는 없다
- 오차항 50% : 우연보다 낫지 않다, 85% : 독립변수가 종속변수에 영향을 미칠 가능성이 상당함

## 회귀분석의 활용
- 현상설명 : 특정 날짜에 판매가 급증하는 이유, 특정 달에 서비스 콜이 증가한 이유, 특정 요일에만 렌트카를 늦게 반납하는 이유(변수) 등
- 예측 : 광고나 판촉 등이 제품 구매에 미칠 영향을 분석해 광고 방향을 설정 등. 보험사나 은행에서 위험분석 및 프리미엄 가격 책정에 자주 활용
- 프로세스 최적화 : 제과점에서 쿠키의 수명과 품질 최적화를 위한 온도 분석 등
- 오류 해결 : 영업시간이 늘면 정량적 매출은 증가하나, 매출비용도 같이 증가하여 이익은 줄어들 수 있음
- 새로운 통찰력 : 대부분의 기업에 쌓여있는 데이터를 이용하여 이때까지 발견하지 못한 정보 산출 가능

## 회귀분석의 과제
- 상관관계는 인과관계와 같지 않아 두 변수 사이의 관계가 반드시 한 변수가 다른 변수의 원인임을 증명하지 못하기 때문에 존재하지 않은 관계를 가정하지 않도록 주의
- 독립변수는 제어하지 못할 수도 있으므로 가장 영향력이 높은 변수가 반드시 제일 중요한 변수는 아님

## GIGO(garbage in garbage out)
- 결과 예측이 정확하려면 데이터가 정확해야 한다
- 데이터가 설명하지 못하는 부분에서 귀중한 통찰을 놓치고 있을 수 있으므로 풍부한 경험에서 나온 직관과 비즈니스 지식으로 검증 필요
